# CLOUD INSTALLATION AND PRICING

Practical guide to deploy this project on `Netlify + Supabase + Groq`.

---

## Stack Overview

- **Netlify**: hosts frontend and serverless API (`/api/chat`).
- **Supabase**: auth + Postgres storage (`chat_sessions`, `chat_messages`).
- **Groq**: LLM inference via OpenAI-compatible API.

---

## Step-by-Step Setup

1. **Create Supabase project**
   - Copy `VITE_SUPABASE_URL` and anon key (`VITE_SUPABASE_PUBLISHABLE_KEY`).
   - Run/apply your chat schema migration so `chat_sessions` and `chat_messages` exist.

2. **Configure Supabase Auth URLs**
   - In Supabase Auth URL settings:
     - Site URL: `https://<your-site>.netlify.app`
     - Redirect URLs: same domain (and custom domain if used).

3. **Create Groq API key**
   - Generate key in [Groq Console](https://console.groq.com).
   - Pick a model, e.g. `llama-3.1-8b-instant`.

4. **Set Netlify environment variables**
   - In Netlify -> Site configuration -> Environment variables.
   - Add all required keys from the section below.

5. **Deploy to Netlify**
   - Connect GitHub repo (or trigger a manual deploy).
   - Deploy and wait for green build.

6. **Run smoke tests**
   - Use the checklist below before sharing the app.

---

## Required Env Vars

Set these in Netlify:

```env
VITE_SUPABASE_URL=https://<project-ref>.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=<supabase-anon-key>

LLM_PROVIDER=groq
GROQ_API_KEY=gsk_xxx
GROQ_MODEL=llama-3.1-8b-instant
```

Common optional vars (if supported in your function):

```env
VLLM_TEMPERATURE=0.7
VLLM_MAX_TOKENS=500
VLLM_TIMEOUT_MS=60000
```

## Smoke Tests (5-10 minutes)

- [ ] Open app; no blank screen or console auth errors.
- [ ] Sign up/sign in works with Supabase auth.
- [ ] Send one chat message; `/api/chat` returns `200`.
- [ ] Response is generated by Groq (no provider key error).
- [ ] New rows appear in Supabase `chat_sessions`.
- [ ] New rows appear in Supabase `chat_messages`.

If a test fails: confirm env vars, auth URLs, then redeploy (Netlify env changes require redeploy).

## Cost Control Checklist

- [ ] Set monthly budget alerts in Netlify, Supabase, and Groq.
- [ ] Keep one default Groq model; avoid expensive model switching.
- [ ] Cap response size (`VLLM_MAX_TOKENS`) to reduce token burn.
- [ ] Add basic per-user/per-IP rate limits in `/api/chat`.
- [ ] Log daily request count, token usage, and error rates.
- [ ] Disable unused preview environments and stale projects.

---

## Go-Live Checklist

- [ ] Production env vars are set and validated.
- [ ] Supabase auth URLs match production domain.
- [ ] RLS enabled for chat tables.
- [ ] Error handling for LLM timeouts/429 responses is user-friendly.
- [ ] Monitoring enabled (Netlify function logs + Supabase logs).
- [ ] Backups/export plan documented for chat data.
- [ ] One rollback path ready (previous Netlify deploy).
